# Record Purpose

1. Read `docs/concepts/repository-purpose.md` when the category boundary or record shape needs clarification.
2. Maintain the sole purpose record at `docs/repository/purpose.md`. Capture author-confirmed outcomes, sub-purposes, and boundaries in concise domain language; place designs, components, and delivery details in their corresponding records.
3. Update affected indexes and `docs/log.md`, then validate the vault.

Finish when exactly one concise purpose file states the high-level reason, sub-purposes, and boundaries, and the author has confirmed any substantive shift.

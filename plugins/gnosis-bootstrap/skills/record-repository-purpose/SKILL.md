# Record Repository Purpose

1. Read `docs/concepts/repository-purpose.md` when the category boundary or record shape needs clarification.
2. Capture author-confirmed intent in `docs/repository/purpose.md`; keep outcomes durable and boundaries explicit.
3. Update affected indexes and `docs/log.md`, then validate the vault.

Finish when purpose, boundaries, and sub-purposes agree and the author has confirmed any substantive shift.

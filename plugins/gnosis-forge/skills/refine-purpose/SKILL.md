# Refine Purpose

1. Read `docs/repository/purpose.md` when it exists and `docs/concepts/repository-purpose.md` when the category boundary or record shape needs clarification. Explore the repository to resolve discoverable facts before questioning the author.
2. Interrogate the author relentlessly until both sides share a precise understanding. Walk every material branch of the purpose tree, including the durable outcome, intended beneficiaries, sub-purposes, boundaries, and dependencies between them. Resolve author-owned decisions one at a time: ask exactly one question per turn, include a recommended answer with its rationale, and wait for the response before continuing.
3. Once no material branch remains unresolved, summarize the proposed purpose and ask the author to confirm that shared understanding has been reached. Do not change repository files before that confirmation.
4. Maintain the sole purpose record at `docs/repository/purpose.md`. Capture only the confirmed outcome, sub-purposes, and boundaries in concise domain language; place designs, components, and delivery details in their corresponding records.
5. Update affected indexes and `docs/log.md`, then validate the vault.

Finish when the author has confirmed the shared understanding and exactly one concise purpose file states the high-level outcome, sub-purposes, and boundaries.

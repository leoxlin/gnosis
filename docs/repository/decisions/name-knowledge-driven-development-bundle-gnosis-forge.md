# Decision

Name the knowledge-driven software-development skill bundle **`gnosis-forge`**. `gnosis-vault` manages and queries knowledge; `gnosis-forge` applies that knowledge to software work and records what the work produces and teaches.

# Why

The former name, `gnosis-bootstrap`, described initialization. Its skills instead support ongoing work from purpose, decisions, repository processes, and implementation truth, plus durable directives for automated agents.

Rejected alternatives:

- **`gnosis-bootstrap`** — rejected because it suggests a one-time setup phase.
- **`gnosis-engineering`** — clear, but less distinctive beside `gnosis-vault`.
- **`gnosis-development`** — literal, but broad and less memorable.
- **`gnosis-praxis`** — avoided because `praxis` is already a sister project.

# Constraints

- Plugin paths and current documentation use `gnosis-forge`.
- The skills retain task-oriented names, including the manually invoked `using-gnosis`, `record-decision`, and `record-directive`.
- “Bootstrap” remains available as a description of the repository's knowledge-first origin, not as the component identity.

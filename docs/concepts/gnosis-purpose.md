# Gnosis Purpose

A **Gnosis Purpose** states an enduring outcome and the boundaries that implementation cannot express fully.

## Use this for

- Durable outcomes, beneficiaries, sub-purposes, and exclusions.

Do not put architecture, plans, milestones, or tasks here.

## Minimum record

- `# Purpose` and `# Boundaries`.
- Optional `# Sub-purposes` when decomposition adds clarity.

## Schema

```yaml
---
type: GnosisPurpose
title: <name>
description: <outcome>
---

# Purpose
# Sub-purposes
# Boundaries
```
